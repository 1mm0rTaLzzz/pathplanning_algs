## Алгоритмы планирования пути

### A*

Алгоритм A* для поиска оптимального пути на графе с эвристической функцией.

**Особенности:**
- Эвристическая функция для ускорения поиска
- Визуализация процесса поиска
- Генерация случайных препятствий
- Гарантированное нахождение оптимального пути

**Использование:**
```bash
python astar_alg/main.py
```

### Dijkstra

Классический алгоритм Дейкстры для поиска кратчайшего пути на графе.

**Особенности:**
- Поиск кратчайшего пути
- Визуализация процесса
- Генерация случайных препятствий
- Гарантированное нахождение оптимального пути

**Использование:**
```bash
python dijkstra_alg/main.py
```

### RRT

Rapidly-exploring Random Tree (RRT) для планирования пути в непрерывном пространстве.

**Особенности:**
- Случайное исследование пространства
- Визуализация дерева и найденного пути
- Работа с непрерывным пространством
- Эффективен в сложных средах

**Использование:**
```bash
python RRT/main.py
```

### RRT*

Оптимизированная версия RRT с гарантированной асимптотической оптимальностью.

**Особенности:**
- Асимптотическая оптимальность пути
- Улучшение качества пути со временем
- Перестроение дерева для оптимизации
- Высокая точность решения

**Использование:**
```bash
python RRTstar/main.py
```

### Потенциальное поле

Метод планирования пути на основе потенциальных полей.

**Особенности:**
- Учет препятствий через отталкивающие силы
- Притяжение к целевой точке
- Плавная траектория
- Работа в реальном времени

**Использование:**
```bash
python potential_field_alg/main.py
```

### Нечеткая логика

Планирование пути с использованием нечеткой логики.

**Особенности:**
- Нечеткие правила для принятия решений
- Учет неопределенности
- Плавное управление
- Адаптивность к изменениям

**Использование:**
```bash
python fuzzy_logic_pathplanning/main.py
```

## Алгоритмы оптимизации

### Генетический алгоритм

Реализация генетического алгоритма для поиска глобальных экстремумов функции.

**Описание:**
Алгоритм решает задачу оптимизации функции:
f(x, y) = - (x² + y - 11)² - (x + y² - 7)²

в области x, y ∈ [-5, 5].

Функция имеет четыре глобальных максимума с одинаковым значением f(x, y) = 0 в точках:
- (3, 2)
- (-2.805, 3.131)
- (-3.779, -3.283)
- (3.584, -1.848)

**Особенности реализации:**
1. **Представление особи**:
   - Вещественное кодирование (x, y в виде чисел с плавающей точкой)
   - Случайная генерация начальной популяции в заданной области

2. **Функция приспособленности**:
   - Значение функции в точке (x, y)
   - Штраф 10% за выход за границы области

3. **Методы селекции**:
   - Турнирная селекция (размер турнира = 3)
   - Рулетка (пропорционально fitness)

4. **Методы кроссовера**:
   - Арифметический (взвешенное среднее)
   - BLX-α (α = 0.5)

5. **Мутация**:
   - Гауссова мутация (σ = 0.5, μ = 0)
   - Адаптивная мутация (увеличение σ при низком разнообразии)
   - Линейное уменьшение вероятности мутации

6. **Элитизм**:
   - Сохранение 5% лучших особей

7. **Критерии остановки**:
   - Максимальное число поколений: 100
   - Остановка при отсутствии улучшений в течение 20 поколений

**Использование:**
```bash
cd genetic_algorithm
pip install -r requirements.txt
python main.py
```

**Результаты:**
После выполнения алгоритма в директории `results` будут сохранены:
- Контурный график функции
- Графики результатов экспериментов
- Графики истории приспособленности для каждого эксперимента
- Графики истории точности для каждого эксперимента

**Анализ результатов:**
Алгоритм проводит эксперименты с различными параметрами:
- Размеры популяции: 50, 100, 200
- Вероятности кроссовера: 0.7, 0.9
- Методы селекции и кроссовера

Для каждого эксперимента измеряются:
- Время выполнения
- Точность решения
- Количество вычислений функции
- Лучшее значение функции

**Структура проекта:**

**Дополнительные задания:**
1. Реализована адаптивная мутация:
   - Отслеживание разнообразия популяции
   - Увеличение σ при низком разнообразии

2. Реализована обработка границ:
   - Ограничение значений в допустимом диапазоне
   - Штраф 10% за выход за границы

3. Многопроцессорная оценка не реализована (опционально)

## Сравнение алгоритмов

| Алгоритм | Оптимальность | Скорость | Память | Применимость |
|----------|---------------|----------|--------|--------------|
| Дейкстра | Оптимальный | Средняя | Высокая | Статические среды, поиск кратчайшего пути |
| A* | Оптимальный при допустимой эвристике | Высокая | Средняя | Статические среды, эффективный поиск пути |
| Потенциальные поля | Не оптимальный | Высокая | Низкая | Динамические среды, локальная навигация |
| RRT | Не оптимальный | Высокая | Средняя | Сложные среды, высокоразмерные пространства |
| RRT* | Асимптотически оптимальный | Средняя | Высокая | Сложные среды, требующие оптимальных путей |
| Нечеткая логика | Не оптимальный | Средняя | Низкая | Управление с неопределенностями, человекоподобное поведение |
